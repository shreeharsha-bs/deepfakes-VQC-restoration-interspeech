Layered Generative Intervention in Deepfake Detection

Decomposing speech authenticity into source identity and processing mediation

Speech authenticity is compositional, not binary. Modern deepfake detectors operate on degrees of algorithmic mediation rather than on an intrinsic notion of realness.

Overview

We challenge the binary "real vs. fake" framing of deepfake speech detection. Speech can undergo multiple layers of generative processing — TTS synthesis, voice conversion, voice quality modification — and each layer adds algorithmic mediation. We define four intervention depths and show that what detectors actually learn is processing depth, not source identity.

Core Finding: The 300× EER Gap

The same model, on the same data, with the same predictions The DF-Arena 4-Way classifier evaluated on ASVspoof5 achieves 0.15% EER when measuring mediation (processed vs. unprocessed) but 46.3% EER when measuring source identity (real vs. fake). This 300× gap proves deepfake detectors learn processing signatures, not authenticity.

EER Analysis

Accuracy can be deceptive DF-Arena binary achieves 94.9% accuracy on ASVspoof5, but its EER is 44.2% — near chance. The high accuracy comes from labeling everything processed as "fake," which works when most ASVspoof5 samples are processed. EER reveals the true real/fake separation capability.

Benign Transformation Analysis

Sidon is a non-adversarial speech enhancement model. Applying it to genuine human speech should not change detector labels — but it does. This demonstrates that detectors are sensitive to any generative processing, not just adversarial manipulation.

Key Hyperparameters

MLP Training

  • Optimizer: Adam (lr=1e-3, default betas)
  • Loss: CrossEntropyLoss
  • Epochs: 100 (early stopping, patience=10)
  • Batch size: 256
  • Dropout: 0.3
  • Hidden dims: 512 → 256
  • Seed: 42

DF-Arena Fine-Tuning

  • Optimizer: AdamW (lr=1e-5)
  • Loss: CrossEntropyLoss
  • Scheduler: Linear warmup + cosine annealing
  • Batch size: 16
  • Max epochs: 10–20
  • Checkpoint: Best validation loss
  • GPUs: Multi-GPU (DataParallel)

Evaluation Protocol

  • Utterance-level splitting: No utterance appears in both train and test.
  • Architecture held-out: OuteTTS is never in training; used only for unseen generalization tests.
  • Cross-domain: ASVspoof5 is completely independent from MLAAD — different speakers, different TTS systems, different recording conditions.
  • EER computation: Equal Error Rate where FAR = FRR, computed from calibrated scores. Used over accuracy as the primary anti-spoofing metric per ASVspoof conventions.
  • Mediation axis: 4-way predictions collapsed to binary (unprocessed vs. processed): {Real, Fake} → Unprocessed; {Real→Conv, Fake→Conv} → Processed.
  • Source axis: 4-way predictions collapsed to binary (real vs. fake): {Real, Real→Conv} → Real; {Fake, Fake→Conv} → Fake.

Key Findings Summary

1. Detectors Learn Mediation, Not Source

The 300× EER gap (46.3% → 0.15%) on the same model proves detectors measure processing depth, not whether audio is human or synthetic.

2. Voice Conversion Collapses Source Identity

Confusion is exclusively between Real→Conv and Fake→Conv. Once voice-converted, the source origin becomes undetectable.

3. Accuracy Is Deceptive

DF-Arena reports 94.9% accuracy on ASVspoof5 but has 44.2% EER — near chance for real/fake separation. High accuracy comes from labeling all processed audio as "fake."

4. Benign Transforms Break Detection

Non-adversarial speech enhancement (Sidon) causes 79.5% of real speech to be labeled as fake by DF-Arena. Any generative processing triggers false positives.

5. Unconverted Detection Is Solved

Distinguishing raw human speech from raw TTS is trivial (99.6% unseen accuracy). The difficulty lies entirely in the converted domain.

6. Cross-Domain Generalization Fails

Models trained on MLAAD achieve 99% in-domain but ~48% on ASVspoof5 for source detection. They learn architecture-specific artifacts, not universal real/fake features.
